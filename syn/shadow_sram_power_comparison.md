## 3. 影子寄存器开销分析 (A1 → B1)

SHADOW_EN=1 使 `regfile` 内部额外实例化了 31 个 32-bit 影子寄存器 (x1-x31 的硬件快照)。

### 3.1 资源增量

| 指标 | SHADOW_EN=0 (A1) | SHADOW_EN=1 (B1) | 增量 |
|------|-------------------|---------------------|------|
| Sequential cells | 19,293 | 20,287 | +994 (+5.2%) |
| Combinational cells | 33,786 | 35,023 | +1,237 (+3.7%) |
| Total cell area | 200,445 | 208,775 | +8,330 (+4.2%) |

### 3.2 功耗增量 (寄存器版 data_ram 下)

| 功耗项 | SHADOW_EN=0 (A1) | SHADOW_EN=1 (B1) | 增量 |
|--------|-------------------|---------------------|------|
| **regfile (寄存器堆)** | 1.925 mW | 3.740 mW | **+1.815 mW (+94%)** |
| core_top 其余 | 2.095 mW | 2.098 mW | +0.003 mW |
| data_ram | 32.060 mW | 32.060 mW | 0 |
| 其他外设 | 1.515 mW | 1.515 mW | 0 |
| **整芯片 Total** | **37.60 mW** | **39.41 mW** | **+1.82 mW (+4.8%)** |

影子寄存器的开销集中在 `regfile` 模块，功耗几乎翻倍 (1.93 → 3.74 mW)，但对整芯片影响仅 4.8%。

### 3.3 功耗增量 (SRAM 版 data_ram 下)

| 功耗项 | SHADOW_EN=0 (A2) | SHADOW_EN=1 (B2) | 增量 |
|--------|-------------------|---------------------|------|
| regfile | 1.925 mW | 3.740 mW | +1.815 mW |
| 整芯片 (含SRAM) | **5.75 mW** | **7.56 mW** | **+1.81 mW (+31%)** |

在 SRAM 版本中，因总功耗基数大幅降低，影子寄存器的相对开销变得显著：**+31%**。

### 关键观察

- **寄存器版 data_ram 统治功耗** (81-85%) → 掩盖了 CPU 核的真正功耗结构
- **SRAM 版让 CPU 核成为功耗焦点**: regfile 占 33-50%, CSR 占 9-12%
- **影子寄存器打开后**, regfile 功耗翻倍 → 在 SRAM 版中变成绝对主导 (50%)

## 7. 综合建议

### 7.1 data_ram 选择

SRAM 替代寄存器阵列收益极大：

| 收益维度 | 无影子寄存器 | 有影子寄存器 |
|---------|------------|------------|
| 功耗 | **↓85%** | **↓81%** |
| 面积 | **↓71%** | **↓68%** |
| 时序 | 需额外1拍读延迟 | 同左 |

**强烈建议使用 SRAM。** 2KB 的 data_ram 用寄存器实现极不经济。

### 7.2 影子寄存器选择

| 指标 | SHADOW_EN=0 | SHADOW_EN=1 | 差异 |
|------|------------|------------|------|
| 功耗 | 5.75 mW (SRAM版) | 7.56 mW (SRAM版) | +1.81 mW |
| 面积 | 58,322 | 66,624 | +8,302 |
| 中断延迟 | 需软件保存/恢复31个寄存器 | **单周期硬件快照** | 节省 ~62 条指令 |

影子寄存器用 **1.81 mW 和 8,302 sq.um** 换取了：
- 中断进入：0 条指令保存上下文 (vs ~31 条 sw)
- 中断返回：0 条指令恢复上下文 (vs ~31 条 lw)
- 中断延迟降低约 **62 个周期** (31 store + 31 load)

对于实时性要求高的应用，这个 trade-off 非常划算。如果功耗是首要约束，可以关闭。

### 7.3 推荐配置

| 场景 | 推荐 |
|------|------|
| 功耗优先 | **A2**: SHADOW_EN=0 + SRAM (5.75 mW) |
| 实时性优先 | **B2**: SHADOW_EN=1 + SRAM (7.56 mW) |
| FPGA 原型验证 | 寄存器版 (FPGA 无 SRAM hard macro) |

## 附录: 功耗分析方法

本次所有功耗数据使用以下设置获得:

```tcl
set power_enable_analysis TRUE       # 必须在 compile 前
set_power_default_toggle_rate 0.1    # 10% 默认翻转率
set_power_default_static_probability 0.5
```

SRAM macro 功耗从 `.lib` 中的 `internal_power()` 表手动计算:

```
能量/操作 (pJ) × 访问频率 (200MHz × 10% × 读/写比例) = 动态功耗 (mW)
漏电功耗直接取自 cell_leakage_power (0.000993 mW)
```

面积从 `.lib` 的 `area` 字段获取 (21,194 sq.um)。
