# 为什么同样 512×32-bit，Via-ROM 面积只有 SRAM 的一半？

**日期**: 2026-06-10
**对比对象**: S55NLLG1PH (SRAM) vs S55NLLGVMH (Via-ROM)，均 512 words × 32 bits

---

## 1. 数据

| 参数 | SRAM | Via-ROM | 比值 |
|------|------|---------|------|
| 存储容量 | 16,384 bits | 16,384 bits | 1:1 |
| Macro 面积 | 21,194 sq.um | 11,295 sq.um | **1.88:1** |

---

## 2. 一句话原因

**SRAM 每个 bit 需要 6 个晶体管做存储+读写，ROM 每个 bit 只需 1 个过孔（有/无）。再加上 SRAM 需要整套写电路而 ROM 根本不需要写。**

---

## 3. 物理结构对比

### SRAM 6T 存储单元

```
     WL (字线)
      │
   ┌──┴──┐
   │     │        M1─M2─M3─M4 构成交叉耦合反相器对（锁存器）
   │  ┌──┴──┐     M5, M6 为传输门
   │  │     │
   └──┤  ┌──┴──┐
      │  │     │
      └──┤     │
         └─────┤
               GND
   ───────────────────────────
   6 个晶体管 / bit
   bit cell 面积 ≈ 0.5–1.0 sq.um
```

### Via-ROM 存储单元

```
     字线 (多晶硅)
        │
        ├────────── 有过孔 = bit=1
        │
        ├────────── 无过孔 = bit=0
        │
     位线 (金属1)
   ───────────────────────────
   1 个过孔 / bit（存在或不存在）
   bit cell 面积 ≈ 0.05–0.1 sq.um
```

---

## 4. 面积分解估算

| 组成部分 | SRAM (sq.um) | Via-ROM (sq.um) | 说明 |
|----------|-------------|-----------------|------|
| **存储阵列** | ~12,000 | ~500 | ROM 阵列密度是 SRAM 的 20+ 倍 |
| **地址译码** | ~2,000 | ~2,000 | 都是 9→512，基本相同 |
| **写驱动电路** | ~3,000 | **0** | ROM 不需要写 |
| **BWEN 逐位控制** | ~1,500 | **0** | ROM 不需要字节使能 |
| **读敏感放大器** | ~1,500 | ~1,000 | ROM 读出更简单 |
| **输出锁存+驱动** | ~800 | ~800 | 基本相同 |
| **其他（控制、电源环等）** | ~394 | ~6,995 | 外围开销 |
| **总计** | **~21,194** | **~11,295** | |

---

## 5. 为什么 ROM 省掉了写电路？

| 功能模块 | SRAM | ROM | 原因 |
|---------|------|-----|------|
| 写驱动 | ✅ | ❌ | ROM 不可写入，出厂即固化 |
| 写入时序控制 | ✅ | ❌ | 不需要控制写脉冲宽度 |
| BWEN 逐位掩码 | ✅ (32路) | ❌ | 不需要部分写入 |
| 写入地址译码 | ✅ | ❌ | ROM 不需要列选通 |
| 写入后读回比较 | ✅ | ❌ | ROM 不需要自检 |
| 交叉耦合反相器 | ✅ (每个bit) | ❌ | ROM 没有锁存器 |

---

## 6. ROM 的局限性

ROM 面积小是付出了代价的：

| 特性 | SRAM | Via-ROM |
|------|------|---------|
| 可写 | ✅ | ❌ 必须预先烧录 |
| 可修改 | ✅ 随时 | ❌ 改一次 = 重新流片 |
| 适合 | 变量、堆栈、Buffer | 指令、常量表、查找表 |
| 密度 | 低 | **极高** |
| 漏电 | ~1 uW | **~0.6 uW** |

---

## 7. 结论

Via-ROM 面积约 SRAM 的一半（21,194 → 11,295 sq.um），原因：

1. **bit cell 密度差异巨大**：6T vs 单过孔，阵列面积差一个数量级
2. **ROM 不需要整套写电路**：写驱动 + BWEN 控制 + 写时序 → 全砍掉
3. **ROM 读出电路更简单**：不需要处理写入破坏读出的边界情况

对 inst_rom 这种"只读指令存储器"，Via-ROM 是天然最优解——面积小、漏电低，唯一的代价是改程序需要重新流片。
